## Known Limitations

1. **Workflow ID Resolution**:
   - Script tries file path first, then numeric ID
   - May require workflow file path instead of workflow name
   - This is a GitHub API limitation

2. **Polling Intervals**:
   - Default 3-second intervals may be too frequent for some use cases
   - Can be adjusted with `--interval` flag
   - GitHub API rate limits apply

3. **Log Availability**:
   - Logs only available after workflow completion
   - Logs expire after 90 days (GitHub retention policy)
   - No real-time streaming (GitHub API limitation)

4. **Correlation Timing**:
   - GitHub may take a few seconds to index new runs
   - Very fast workflows might complete before correlation succeeds
   - Polling mechanism handles this with retries
